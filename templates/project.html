Tutor Management SystemJan. 2016

1. Intention

  • I have been tutoring quite a lot and found it's quite inconvenient to manage so many students through phone call or text message. So I made this website to allow students book my tutoring online and thus making my life easier.

2. Functionality and Features

  • A website built on Google App Engine
  • Window size responsive, which means can be both used with laptop or mobile devices
  • Account management allows students create their account and book tutoring with their personal account.
  • Safe encrypted passwords and cookies make sure students account do not get hacked
  • Money topped up through back-end database manipulation

3. Functionalities to be added

  • Better student-end account management including: change password, reset password, cancel reservations and so on
  • Database transaction docs which tracks database changes and allow restoring database
  • Management GUI for the tutor(i.e me or any other tutor who want to use this app), including functions like create and adjust a class session, adjust students' reservations and so on
  • Forum for students' discussion
  • Students' feedback to the tutor
  • ...

4. Technologies Involved

HTML, CSS, Javascript, jQuery, Ajax, Database, Google App Engine, Bootstrap

TA-Helper Web ApplicationJan. 2016

1. Intention

  • It can help teaching assistants to manage a lab session especially in which every student has a computer such a programming lab session
  • Make students fairly and easily queue for helps from TA and free them from constantly checking if a TA is available
  • Save students' time by let TA first help students with short questions which can be solved in a couple of seconds. Basically, students with short questions have higher priority over students with questions taking longer time(such as debugging a program). In this way, it's efficient for the whole class.

2. Functionality and Features

  • Four Pages:
    • Student's Main Page: for student to choose a session
    • Student's Queue Page: students make queue operation in this page
    • TA's main page: for TA to choose a session to manage as well as create a new session
    • TA's managing page: in a particular session, the TA can check the queue and remove students from queue when problems are solved
  • Multiple sessions can work at the same time
  • TA set the session time and the sesion auto-vanish after the setted end time
  • Students can queue and dequeue themselves from students end. TA can also dequeue students from TA end. Those process are all real time

3. Technologies Involved

HTML, CSS, Javascript, jQuery, Ajax, Database, Google App Engine

Research on Multi-Channel Payment System for Vending Machines Dec. 2014 - May. 2015

1. Intention

  • Traditional vending machines allows cash-only payment, which does not go with the fact that student card is widely used for payment on campus.
  • Try to combine the vending machines on campus and the student card and achieve the function of purchasing goods from vending machine with student card.
  • From the students' side, this project means more convenience.
  • From the vendors' side, this project may increase their selling

2. Functionality and Features

  • The project succeeded and managed to realize the payment with student card.
  • Single-chip powered serial communication between the vending machine and the POS machine which handles transactions of student cards.

3. Award

  • Best Thesis/Design (Zhejiang University 2015) (top 10%)
